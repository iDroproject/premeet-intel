name: Data Terminology
description: Use "person data"/"company data" instead of "enrichment" for LinkedIn/web scraping data
type: feedback
---

Reserve "enrichment" for additive data operations (e.g., enrichment-mcp adding Crunchbase/ZoomInfo data on top of base profiles). Use "person data" or "company data" when referring to LinkedIn/web scraping via BrightData.

**Why:** The CEO corrected this to maintain clear terminology boundaries in the codebase and product language.

**How to apply:** In code comments, PR descriptions, task updates, and UI copy — use the correct term based on the data source and operation type.
